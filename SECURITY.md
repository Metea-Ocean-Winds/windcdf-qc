# Security Policy

## Supported Versions

WindCDF is a scientific and engineering tool primarily intended for offline
data analysis. Security updates are provided on a best-effort basis for the
latest released version.

---

## Reporting a Vulnerability

If you discover a potential security vulnerability, please **do not open a
public issue**.

Instead, report it responsibly by:

- Contacting the maintainers via GitHub (private message), or
- Sending a detailed report to the email address listed in the repository
  contact information (if available)

Please include:
- A clear description of the issue
- Steps to reproduce (if applicable)
- Potential impact

---

## Scope of Security Concerns

Relevant security issues may include:
- Arbitrary code execution
- Unsafe file handling
- Malicious data injection through NetCDF inputs

Issues such as incorrect QC results or scientific disagreements should be
reported as **bugs or discussions**, not security vulnerabilities.

---

## Disclosure Policy

We aim to:
- Acknowledge receipt of a vulnerability report promptly
- Assess and address valid issues in a reasonable timeframe
- Coordinate responsible disclosure where appropriate
